# Lanzando excepciones

En cualquier lenguaje de programación, los errores son inevitables. Hay (al menos) dos tipos diferentes de errores: errores de sintaxis y excepciones.

Los errores de sintaxis son errores en la estructura del código, como olvidar un paréntesis o una comilla. Estos errores son detectados por el intérprete de Python antes de que el programa se ejecute.

Las excepciones, por otro lado, son errores que ocurren durante la ejecución del programa. Cuando una excepción no se captura, se produce una interrupción abrupta del programa, por lo que son eventos que alteran el flujo normal de un programa. Estos errores pueden ser causados por una variedad de razones, como intentar dividir por cero, acceder a un índice fuera de los límites de una lista o intentar convertir una cadena que no representa un número en un entero.

Cuando se produce (o se lanza) una excepción, Python genera un objeto de excepción que contiene información sobre el error. Si no se captura la excepción, el programa se detiene mostrando un mensaje de error.
